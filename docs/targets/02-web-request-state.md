# 目标 02：收敛 Web 请求状态

## 目标
整理内置 Web 前端的 API 请求、认证失败和错误渲染路径，使配置、渠道测试、自拍和记录操作使用一致的状态更新规则。

## 范围
- 抽取重复的前端请求及错误处理逻辑。
- 明确登录失效、网络错误、服务端错误和任务轮询失败的界面状态。
- 使用 Flask test client 覆盖对应 API 错误响应契约。

## 非目标
- 不引入 npm、Vite、React 或其他构建链。
- 不改变 Web Token 的空值免校验策略。

## 验收
- 前端不重复拼装鉴权头和 JSON 错误信息。
- 失败时保留当前表单数据，不把错误响应直接插入 HTML。
